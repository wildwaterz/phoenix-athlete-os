# Roadmap

Version: 1.0
Owner: Kevin Sauvageau
Product: Project Phoenix OS
Status: Living Document

## Purpose

This roadmap defines how Project Phoenix OS should evolve.

The goal is to protect focus and avoid building too much too early.

## Sprint 0: Product Foundation

Status: In progress

Goals:

- Define product vision
- Create project documentation
- Establish MVP scope
- Validate core UX direction

Key outputs:

- PROJECT.md
- Product principles
- Coach packet schema
- Decision engine
- Recovery IQ rules

## Sprint 1: Daily Loop MVP

Goal:

Make Project Phoenix useful every day.

Scope:

- Dashboard
- Daily Check-In
- Date selector
- Morning and evening check-ins
- Coach Packet export
- Coach Journal
- Recovery IQ
- Missions
- Milestones

Success criteria:

- Daily check-in takes less than two minutes
- Coach packet requires no manual cleanup
- Dashboard clearly answers what to do today

## Sprint 2: Data Quality and Persistence

Goal:

Ensure data can be stored, reviewed, and exported reliably.

Scope:

- Local persistence or Supabase
- Historical entries
- Backfill missed days
- Trend calculation
- Export all data

## Sprint 3: Analytics and Evidence Timeline

Goal:

Help the athlete understand progress over time.

Scope:

- Trend charts
- Evidence timeline
- Decision history
- Milestone unlock history
- Readiness trends

## Sprint 4: Training and Capacity Module

Goal:

Track strengthening, conditioning, and load tolerance.

Scope:

- Exercise sessions
- Load tracking
- Pain during/after
- Next-day response
- Capacity milestones

## Sprint 5: Pre-Surgery Readiness Dashboard

Goal:

Prepare for definitive ACL revision surgery.

Scope:

- Readiness score
- Mobility status
- Strength status
- Conditioning status
- Body composition inputs
- Coach summary export

## Future Campaigns

Potential modules:

- ACL revision rehab after definitive surgery
- Hockey readiness
- Strength training
- Golf performance
- Conditioning
- Nutrition
- Longevity
- Wearable integrations
