# Product Backlog

Version: 1.0
Owner: Kevin Sauvageau
Product: Project Phoenix OS
Status: Living Document

## Purpose

This backlog organizes product work by priority.

The goal is to keep Project Phoenix focused and prevent feature bloat.

These features are required for daily use.

- Current mission
- Date selector
- Recovery day
- Today’s readiness
- Key metrics
- Mission progress
- Today’s quests
- Coach recommendation
- Coach journal preview
- Recovery phase indicator

### Daily Check-In

- Morning check-in
- Evening check-in

- Form sections

### Coach Packet Export

- Morning packet
- Evening packet
- Markdown export

### Missions
